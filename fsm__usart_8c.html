static void do_set_data_tx ( fsm_t *  p_this)

Set the data to be sent.

This function sets the data to be sent by the USART to the internal buffer of the PORT layer.

TO-DO alumnos:

✅ 1. Call the right function to reset the output data in the PORT layer before setting a new one
✅ 2. Call function port_usart_copy_to_output_buffer() to set the data stored in the array out_data. The length of the data are defined by USART_OUTPUT_BUFFER_LENGTH
✅ 3. Call function port_usart_write_data() to write the first character to the USART TX register
✅ 4. Call function port_usart_enable_tx_interrupt() to enable the USART TX interrupt to start the transmission


Note
The order of these two function calls is important because of how the USART HW and its interrupts work.

    💡 1. port_usart_write_data() writes the first character to transmit. Once the data is written to the DR register, the USART hardware starts transmitting the data.
    💡 2. port_usart_enable_tx_interrupt() enables the USART transmit data register empty (TXE) interrupt. The TXE interrupt is triggered when the TX register is empty, i.e., when the data in the DR register has been moved to the shift register for transmission.
     If you enable the TXE interrupt before writing the first character to the DR register, the TXE interrupt might be triggered immediately because the TX register is initially empty. This could result in an interrupt routine being called before you expect it, which might cause problems if the interrupt routine assumes that the first character has already been started to transmit.
     By writing the first character to the DR register before enabling the TXE interrupt, you ensure that the TXE interrupt is not triggered until after the first character has been started to transmit. This makes the control flow of your program more predictable and helps to prevent potential race conditions or timing issues.

◆ fsm_usart_new()

fsm_t* fsm_usart_new ( uint32_t  usart_id)

Create a new USART FSM.

This FSM implements a USART communication protocol. It is a state machine that sends and receives data.

The FSM stores the received data in the in_data array. The user should ask for it using the function fsm_usart_get_in_data().

At start and reset, the in_data array must be empty. An empty array means that there has not been new data.

Attention
The user is required to reset the in_data array once it has been read. Otherwise, this value may be misinterpreted by the user, if successive calls are made without having received new data. In such a case we would be reading past information. In order to reset the value, the function fsm_usart_reset_input_data() must be called.

In other words, the status flag of this FSM is the variable in_data. An empty array means that no new data has been received, a value other than 0 means that it has been received and the value is its duration, so it is the user's responsibility to clear this status flag.

The FSM contains Information of the USART ID. This ID is a unique identifier that is managed by the user in the port. That is where the user provides identifiers and HW information for all the USARTs on his system. The FSM does not have to know anything of the underlying HW.

A pointer to the USART FSM
