Set the duration of the timer that controls the duration of the note.

This function is called by the _start_note() private function to set the duration of the note in milliseconds.

Because there could be several notes playing at the same time, each buzzer melody player has its own timer. So, the timer to be configured is selected by the buzzer_id parameter, and you must encapsulate the configuration of the timer in a switch or if statement.

Use the timer indicated in the main page.

    ðŸ’¡ This is a temporary value because it could be greater than 65536 (or 0xFFFF)
âœ… 3. Compute the prescaler value and set the PSC register of the timer.
    ðŸ’¡ Compute a temporary value of the prescaler starting with 0 and increase it until the value of the ARR register is less than 65536 (or 0xFFFF)
âœ… 4. Set the ARR register of the timer with the temporary value computed in step 2, and set the PSC register with the temporary value computed in step 3
âœ… 5. The PSC and ARR values are currently in the preload registers. To load them into the active registers we need an update event. We achieve this by setting the UG bit of the EGR register. It is important to do this at the end of the configuration of the timer.
